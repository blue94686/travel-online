# 后台爬虫补全与后台数据呈现优化设计

## 背景

后台已有资料补全、公开来源采集、A 级源表慢速补图任务和候选审核能力。当前需要把剩余缺图景区持续补全，并扩展到景区介绍、周边美食 POI、徒步/步道 POI，同时让管理员可以手动启动、停止、刷新和批量处理低风险结果。

## 目标

- 管理员可在后台手动启动一个慢速爬虫补全任务，逐步处理剩余缺图和缺资料景区。
- 外部采集结果默认进入候选池，不直接覆盖正式数据。
- 图片外链和低风险 POI 支持一键批量通过，并写入正式景区资料。
- 后台能实时展示任务进度、剩余缺图、候选数量、低风险可通过数量、来源冷却和失败摘要。
- 优化后台管理的数据呈现：让数据中心、资料补全、爬虫任务、质量指标更清晰、更适合管理员反复操作。

## 非目标

- 不下载和托管外部大图，本轮继续保存外链、来源、许可证和署名。
- 不绕过候选池直接覆盖介绍类文本。
- 不新增大型调度系统；本轮沿用现有后台任务表和轻量 worker 模式。
- 不做整个后台视觉品牌重构，只做信息架构、密度、状态表达和操作区优化。

## 推荐方案

采用“任务化爬虫 + 候选池 + 批量低风险通过”。

爬虫任务读取缺图、缺介绍、缺周边数据的景区，按公开来源和已配置地图来源采集候选。采集结果写入现有候选模型或扩展候选类型。管理员在后台查看进度，手动批量通过低风险图片外链和 POI。文本介绍、来源风险较高或匹配不确定的结果继续人工审核。

## 后端设计

### 接口

- `POST /api/admin/enrichment/crawler/start`
  启动慢速补全任务，参数包括批次大小、最大处理数、是否只处理缺图、是否采集 POI、是否启用付费/配额型地图来源。
- `GET /api/admin/enrichment/crawler/status`
  返回任务状态、运行进度、剩余缺图、候选统计、低风险可通过统计、最近失败、来源冷却时间。
- `POST /api/admin/enrichment/crawler/stop`
  请求停止任务，当前批次完成后停止。
- `POST /api/admin/enrichment/crawler/batch`
  执行一次同步批次，方便管理员小批量试跑。
- `POST /api/admin/enrichment/crawler/approve-low-risk`
  批量通过低风险图片外链和 POI 候选。

### 候选类型

- `summary` / `description`: 景区介绍候选，默认人工审核。
- `image`: 图片外链候选，低风险可批量通过。
- `nearby_food`: 周边美食 POI 候选，低风险可批量通过。
- `hiking_poi`: 徒步/步道 POI 候选，低风险可批量通过。
- `nearby_poi`: 其他周边 POI 候选，默认候选审核。

### 写入策略

- 采集阶段只写候选池、任务状态和审计日志。
- 批量通过低风险图片时，写入 `scenic_images`，必要时补 `scenic_spots.cover_image_url`。
- 批量通过低风险美食 POI 时，合并写入 `scenic_spots.nearby_food`。
- 批量通过低风险徒步 POI 时，合并写入 `scenic_spots.nearby_pois` 和 `scenic_spots.recommended_routes`。
- 合并 JSON 字段时按名称、来源、坐标去重，保留已有人工数据。

## 爬虫来源与风险

- 公开介绍和图片：Wikipedia、Wikivoyage、Wikimedia Commons。
- 徒步/步道：OpenStreetMap Overpass，重点筛选 trail、path、viewpoint、peak、route 等公开地理要素。
- 周边美食：优先使用已配置地图服务；未配置时生成可审核的本地规则候选，不伪造具体商户。
- 低风险判定：公开来源、坐标距离合理、名称与景区或城市匹配、来源 URL 可追溯、图片带来源/许可信息。

## 前端后台设计

### 资料补全中心

新增“爬虫补全”主卡片：

- 顶部显示运行状态、剩余缺图、候选总数、低风险可通过数。
- 中部显示进度条、当前批次、最近更新时间、来源冷却倒计时。
- 操作区提供开始慢速补全、试跑一批、刷新、停止、批量通过低风险。
- 底部显示最近失败和来源健康状态。

### 后台整体排版优化

- 数据中心首页改成“关键指标 + 任务状态 + 待办审核 + 最近批次”的层次。
- 卡片统一使用紧凑指标、状态徽标和清晰按钮分组，减少长段说明。
- 对后台数据表格增加更稳定的列宽、空状态、更新时间、来源标签和风险标签。
- 资料补全相关信息集中展示，避免图片任务、候选采集、源表补齐散落在多个区域。
- 移动端保持可读，按钮自动换行但不挤压文字。

## 错误处理

- 来源限流时进入冷却并继续记录状态，不把任务标记为失败。
- 单个景区采集失败只记录失败摘要，任务继续处理后续景区。
- 外部来源不可用时显示来源状态，并允许管理员稍后手动重试。
- 批量通过时跳过已失效、重复或高风险候选，并返回跳过原因统计。

## 测试计划

- 后端单测覆盖爬虫批次候选写入、状态统计、低风险批量通过和 JSON 合并去重。
- 前端测试覆盖后台爬虫卡片状态、按钮调用、运行中/冷却/失败文案和批量通过结果。
- 运行现有资料补全、媒体任务、后台页面相关测试，确认现有接口不回归。

## 验收标准

- 管理员能启动、停止、刷新爬虫补全任务。
- 后台能看到剩余缺图数量、任务进度、候选数量和低风险可通过数量。
- 爬虫采集的介绍、图片、周边美食 POI、徒步 POI 默认进入候选池。
- 低风险图片外链和 POI 可以一键批量通过并写入正式数据。
- 后台数据中心和资料补全中心排版更清晰，关键数据无需滚动大量文字即可看见。
